# 网关与混合推理

网关推理允许应用通过 HTTP 调用独立 CogentLM 网关。混合推理指在同一个客户端中同时注册本地和网关端点，每次请求按需选择执行位置。

## 何时使用网关

- 避免在浏览器或边缘客户端中暴露云端服务商凭证。
- 集中管理访问策略和并发限制。
- 在受控服务器上提供本地模型服务。
- 为多种语言环境提供稳定一致的 HTTP 边界。

## 网关部署形式

官方网关支持三种部署形式：

- **板载 GPU 推理**：网关加载本地 GGUF 模型，利用 GPU 后端提供服务。
- **纯服务商路由**：网关不加载本地模型，将请求转发给外部服务商（如 OpenAI、Anthropic 或兼容 OpenAI 的 API）。
- **混合模式**：网关同时提供本地 GPU 目标和外部服务商目标。

## 端点模型

客户端不会自动路由请求。应用必须显式注册端点描述符，在请求时指定端点引用：

- **本地描述符**：当前进程直接加载 GGUF 模型。
- **网关描述符**：包含网关基础 URL、目标名称、路由路径和身份验证信息。
- **服务商描述符**：客户端直连云端服务商的适配器。

网关描述符将目标名称映射到配置中的 `model` 字段。网关接收请求后，将该名称解析为具体本地模型或服务商端点。

## 身份验证

服务器和脚本环境通过环境变量读取 Bearer 凭证。浏览器应用应在运行时通过 Token 回调动态提供临时凭证。
