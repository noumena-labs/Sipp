# 视觉模型

视觉工作流允许在发送聊天消息时附加图像数据。目前仅支持本地运行，除非应用网关专门实现了对应的媒体支持协议。

## 本地视觉

运行本地视觉模型通常需要：

- 兼容且具备视觉处理能力的 GGUF 模型文件。
- 模型架构有要求时，还需投影器 GGUF 文件。
- 图像的本地路径、浏览器 Canvas 导出数据或图像字节数据。

Rust、Node.js 和 Python 的示例目录中都包含 `vision_chat` 示例代码。浏览器演示项目也展示了基于 Canvas 和本地文件的相关工作流。

## 浏览器视觉

浏览器应用应确保捕获的图像载荷尽可能小，并严格聚焦于任务。例如，主动绘图演示项目只使用裁剪后的 JPEG 捕获图像，确保模型只能"看"到相关墨迹，而非整个页面截图。
