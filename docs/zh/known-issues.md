# 已知问题

使用 Sipp 时常见的已知问题和规避方案。

## 浏览器Pulse Animation可能降低 WebGPU 解码吞吐量

状态：待解决

网页中持续渲染的呼吸（pulse）动画会拖慢浏览器本地推理性能，尤其影响 WebGPU 的解码吞吐量。模型生成阶段如果页面仍在绘制呼吸 UI 或场景特效，每秒生成 Token 数会显著下降。

受影响场景：

- 用 `@sipphq/sipp` 浏览器包执行本地推理。
- 演示项目或应用在模型解码阶段运行复杂呼吸动画。

临时方案：

- 触发请求解码时暂停或移除呼吸动画。
- 内容生成期间优先用静态指示器或降低 UI 刷新频率。
- 横向对比各类后端或模型的吞吐量时，务必关闭视觉动画。

## 双显卡笔记本可能默认调用集成显卡

状态：待解决

同时配备集成显卡和独立显卡的 Windows 笔记本上，浏览器有时会默认分配集成显卡运行 WebGPU。本地推理不会中断，但解码吞吐量会远低于独立显卡的正常水平。

解决方案：

1. 打开 **Windows 设置**。
2. 进入 **系统 > 屏幕 > 图形**。
3. 手动添加运行 Sipp 的浏览器可执行文件（Chrome、Edge 或其他 Chromium 内核浏览器）。
4. 将该浏览器的图形首选项设为**高性能**。
5. 重启浏览器，重新加载 Sipp 页面。

系统级设置比浏览器内部标志（flags）更可靠，它直接强制 Windows 为浏览器进程分配性能最高的 GPU。
